# 학습정리
## JAVA keyword -> 53개

### 데이터 타입과 관련된 키워드
- byte: 8비트 부호 있는 정수형 데이터 타입
- short: 16비트 부호 있는 정수형 데이터 타입
- int: 32비트 부호 있는 정수형 데이터 타입
- long: 64비트 부호 있는 정수형 데이터 타입
- float: 32비트 부동 소수점 실수형 데이터 타입
- double: 64비트 부동 소수점 실수형 데이터 타입
- char: 16비트 유니코드 문자형 데이터 타입
- boolean: true 또는 false 값만 가지는 논리형 데이터 타입
- true: boolean 데이터 타입의 true 값을 나타내는 예약어
- false: boolean 데이터 타입의 false 값을 나타내는 예약어
- null: 값이 없음을 나타내는 특수한 참조 타입 값. 모든 참조 변수의 기본값

### 제어문과 관련된 키워드
- if: 조건문
- else: if문 이외의 조건
- switch: 다중 분기문
- case: switch문에서 조건
- default: switch문에서 case에 해당하지 않는 경우
- while: 반복문
- do: while문과 함께 사용되며, 조건 검사보다 먼저 실행
- for: 반복문
- break: 반복문이나 switch문을 빠져나갈 때 사용
- continue: 반복문의 현재 반복 종료, 다음 반복 시작
- *goto: 미사용. 다른 언어에서는 프로그램 흐름을 지정된 라벨로 이동시키는 데 사용될 수 있는 예약어

### 예외 처리와 관련된 키워드
- try: 예외가 발생할 가능성이 있는 코드 블록
- catch: try 블록에서 예외가 발생하면 해당 예외를 처리하는 코드 블록
- finally: 예외 발생 여부와 상관없이 항상 실행되는 코드 블록
- throw: 예외를 발생
- throws: 메소드에서 발생할 수 있는 예외
- assert: 주어진 조건이 만족되지 않으면 AssertionError 발생

### 클래스와 객체와 관련된 키워드
- class: 클래스 정의
- new: 객체 생성
- this: 객체 자신 참조
- super: 부모 클래스 참조
- extends: 클래스 상속 표현
- instanceof: 객체의 타입 비교
- final: 불변 상태
- abstract: 추상 클래스나 추상 메서드 정의
- enum: 열거형 타입 정의

### 메소드와 관련된 키워드
- void: 해당 메소드가 반환값을 가지지 않음
- return: 메소드에서 값을 반환할 때 사용
- public: 해당 메소드를 어디서든 호출 가능한 공개 메소드로 선언, 다른 패키지에서도 접근이 가능
- private: 해당 메소드를 같은 클래스 내에서만 호출 가능한 비공개 메소드로 선언, 다른 클래스에서는 접근이 불가능
- static: 해당 메소드가 인스턴스화하지 않고도 클래스 이름으로 직접 호출할 수 있는 정적 메소드로 선언
- protected: 해당 메소드를 같은 패키지 내의 클래스와 상속받은 클래스에서 접근할 수 있는 protected 메소드로 선언, 그 외의 외부 클래스에서는 접근이 불가능
- native: 자바 프로그램 안에서 네이티브 코드(Java 바이트 코드가 아닌, 다른 언어로 작성된 코드)를 호출용도


### 인터페이스와 관련된 키워드
- interface: 인터페이스를 정의
- implements: 인터페이스 구현을 표현

### 패키지와 관련된 키워드
- package: 패키지 선언 키워드입니다. 해당 클래스가 속한 패키지를 명시
- import: 패키지를 가져오는 키워드입니다. 다른 패키지에 있는 클래스나 인터페이스를 사용할 때 필요

### 기타: synchronized, volatile, transient, native 등
- 변수 선언
    - *const: 미사용. 상수 선언 키워드로, 값을 변경할 수 없는 변수를 정의할 때 사용
    - transient: 직렬화 시 제외하고 싶은 멤버 변수 표시, 직렬화된 객체를 다시 복원할 때 해당 변수는 초기화되어 저장않음
    - volatile: 멀티 쓰레드 환경에서 공유되는 변수를 정의할 때 사용합니다. 해당 변수가 변경될 때마다 메인 메모리에서 값을 읽어오도록 보장하며, 쓰레드 간의 일관성을 유지
- 부동소수점 연산
    - strictfp: 부동 소수점 연산에서 IEEE 754 규격을 따르도록 강제, JVM에서 실수형 연산이 일관된 방식으로 수행될 수 있도록 한다.
- 동시성 제어
    - synchronized: 멀티스레딩 환경에서 공유 자원에 대한 동시 접근을 제어하기 위한 키워드로, 한 스레드가 해당 자원을 사용 중이면 다른 스레드는 대기하게 되어 동시 접근을 막는다.

## 1장

### 자바
- 자바는 `객체`지향 프로그래밍을 지원하는 언어이다.
- 자바 프로그램 개발에 사용하는 툴을 `JDK`라고 한다.

### 자바의 동작
- 사람이 문자를 나열해서 작성하는 프로그램을 `소스코드`라고 하며, 이것을 저장하는 파일에는 `*.java`라는 확장자를 붙인다.
- `*.java`는 바로 실행할 수 없으므로 `컴파일`작업이 필요하며 이때 필요한 것이 `javac`명령이다.
- `컴파일`의 결과로 생성되는 바이트코드를 저장하고 있는 파일을 `클래스 파일`이라고 하며, 확장자는 `*.class`이다.
- 생성된 파일 내에 담겨 있는 `클래스` 를 실행하려면 `java` 명령을 실행한다.

### 주석
- 작성자를 포함해서 코드를 읽는 사람에게 전달하고 싶은 정보를 프로그램 내에서 간단히 기술하는 것을 `주석`이라 한다.
- 이것의 유/무나 내용에 따라 프로그램 동작이 바뀌는 경우는 `없다`.
- 전동적인 `주석`은 `/*`과`*/`로 감싸며, 문서화 `주석`은 `/**`와`*/`로 감싼다. 두 방식 모두 시작 기호와 종료기호가 동일 행 안에 위치`하지 않아도 된다`.
- 한 줄 `주석`은 `//`부터 해당 줄의 마지막까지 주석이 된다. 이 형식은 여러 줄에 걸쳐 사용할 수 `없다`.

- "ABC" 처럼 큰따옴표로 감싸 문자들을 나열한 것을 `문자열 리터럴`이라고 한다.

- `System.out.println("AB\nC"+"D\n\nEFG");`를 실행하면 다음과 같이 표시된다.
    ```
    AB
    CD

    DFG
    ```
### java code
```
public class WindWoodFireMountain{
    public static void main(String[] args){
        System.out.println("금");
        System.out.println("상");
        System.out.println("첨");
        System.out.println("화");
    }

```

- 기호의 명칭
    - `;` 세미콜론
    - `:` 콜론
    - `.` 닷, 포인트
    - `,` 컴마
    - `{` 중괄호
    - `(` 소괄호
    - `'` 작은따옴포
    - `"` 큰따옴포

    
## 2장
### `기본형`
- `정수`: byte, short, int, long
- `실수`: float, double
- `문자`: char
- `논리`: boolean

### 변수
- 숫자 등의 데이터를 자유롭게 저장했다가 꺼낼 수 있는 것이 `변수`이다. 
- 필요한 시점에 `자료형`에 이름을 부여해서 선언해야한다.
- 생성시 값을 넣는 것을 `초기화`라고 하며, 한 번 생성한 후에 값을 넣는 것을 `대입`이라고 한다.

### 연산자
- 계산하는 +나 * 등의 기호를 `연산자`라고 하며 연산 대상이 되는 식을 `연산식`라고 한다.
- 연산의 대상이 되는 `피연산자`의 개수는 `연산자`에 따라 다르다.
- 적은 것은 `1`개이고 많은 것은 `3`개 이다.

### 들여쓰기
- 소스 프로그램은 그 구조에 맞게 왼쪽에서부터 공백을 두고 작성하면 읽기 쉬워진다. 이 공백을 `들여쓰기`이라고 한다.

## 3장 프로그램의 흐름:분기

### 관계연산자
- 대소관계를 판정하는 연산자 -> 결과가 논리값 리터럴

### `표현식`(expression)
- 변수나 리터럴 또는 변수와 리터럴을 연산자로 결합한 것.
- 세미콜론이 붙이면 `표현문`이 되며, 세미콜론만 있는 문을 `공문`이라고 한다.

### `블록`
- 하나 이상의 문을 `{}`로 감싸는 문

### 비교
- 피연산자의 대소 관계를 판정하는 연산자 `<`, `>`, `<=`, `>=` `관계연산자`
- 같은지를 판정하는 연산자 `==`, `!=` `등가연산자`
- 연산자 `&&`, `||`는 `논리연산자`이며, 연산자 `!`는 `논리부정연산자` `A?B:C`는 `삼항연산자` 라고 한다.

### `단락평가`(short circuit evaluation)
- 논리 연산자의 평가결과가 왼쪽 피연산자만으로도 명확한 경우에는 오른쪽 피연산자는 평가하지 않는 것
- 논리합 || : false || true || false
    - 2번의 true가 나온 즉시 평가 종료 true 반환
- 논리곱 && : false || true
    - 1번의 false가 나온 즉시 평가 종료 false 반환

### 기타
- `예약어`, `키워드` if나 else처럼 특별한 의미가 부여된 어구
- 변수나 메서드 등에 부여하는 이름 `식별자`
- 변수의 값이 작은 것부터 큰 순서로 나열하는 것을 `정렬`이라고 한다.
- 알고리즘이란 문제를 해결하기 위한 것으로 `명확`하게 정의돼 있으며, 순서가 정해져 있는 `유한개`의 규칙의 집합이다.
